02 Three repo roles

Every repo is exactly one of these. If it doesn't fit, it doesn't ship.

aux-frameworksDefinition

The standard.

The category-anchor repo. Heuristics, stages, taxonomies — as YAML. This is the Wikipedia page for agentic UX.

contains: aux-heuristics.yaml · trust-architecture.yaml · evolution-curve.yaml
aux-auditExecutable

The grader.

CLI that runs the frameworks against an agent spec. Emits score, grade, named violations, and CI-ready SARIF.

npx aux-audit run ./spec.yaml
trust-gap-classifierExecutable

The detector.

Scans agent interaction logs for named trust failures. Turns "it feels off" into a labelled, filed gap.

npx trust-gap-classifier scan ./logs/*.jsonl
agent-memory-policyExecutable

The bridge.

One config → policy doc + system-prompt snippet + compliance JSON. Product, legal, dev — same source.

npx agent-memory-policy gen ./memory.yaml
agent-ux-teardownsReference

The receipts.

Structured teardowns of shipped products. analysis.md + score.json, schema-validated. Comparable, citable.

/teardowns/<product>/ · score.json
confidence-cues-ui-kitReference

The demo.

One pattern, runnable, clean. "Here's what good looks like." Mini product, not a tutorial.

demonstrates: aux.H06 Graceful Uncertainty
■ Definition = category ownership ■ Executable = dev respect ■ Reference = show, don't tell

03 Workflow · how it runs end-to-end

The canonical flow: an agent spec goes in, a graded scorecard with named gaps comes out, and the fix pattern is already named.

Input

agent-spec.yaml

Surface, autonomy, memory, tools, flows, guarantees, golden transcripts.
→
Process

aux-audit + classifier

Runs heuristics + trust stages. Replays golden transcripts. Labels gaps against taxonomy.
→
Output

scorecard + fix plan

Score, grade, trust stage, evolution stage, named gaps, fix patterns — all reproducible.
score: 72
grade: B
trust_stage: contextual
evolution_stage: task-aware
issues:
  - id: tg.judgment.overreach_in_ambiguity
    severity: high
    evidence: "agent executed multi-step plan with no preview"
    fix_pattern: intent-handshake
recommendations:
  - "Add intent-handshake before multi-step tool use"
  - "Differentiate gated vs autonomous by user tenure"

04 Trust ladder · the scoring axis

Every audit places the product on one of four rungs. Trust is built sequentially — and a violation at stage 4 snaps all the way back to stage 1.

aux.T01

Functional

Can it complete basic tasks reliably?

fails on: hallucination · silent_degradation
aux.T02

Contextual

Does it remember me?

fails on: memory_amnesia · preference_ignored
aux.T03

Judgment

Does it call it right in the ambiguous case?

fails on: overreach · refusal_when_escalation_needed
aux.T04

Advocacy

Does it act for me when incentives misalign?

fails on: metric_over_user · incentive_misalignment
